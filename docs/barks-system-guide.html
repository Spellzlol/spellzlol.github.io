Система баркoв
после рефакторинга

Руководство для дизайнеров и нарративщиков, работающих в Unreal Editor. Объясняет, как теперь запускать реплики NPC: через реактивную очередь или прямое скриптовое воспроизведение — и что больше не нужно трогать на Blueprint NPC.

00 ·

Коротко

Главное
  • Контент барков (строки, VO, правила подбора) по-прежнему живёт в UBarkDefinitionDataAsset — ассеты DA_*_Barks на каждого NPC. Это не менялось.
  • Теперь есть два чётких пути запуска: реактивный (очередь, все NPC слушают) и скриптовый (прямое воспроизведение на конкретном NPC).
  • С NPC Blueprint убраны путаные runtime-поля и функции: StoredBarkTrigger, BarkTrigger, StoredBarkCustomTrigger, StoreBarkData, CommitBark, CommitBarkWithStoredData, bAutoSetTargetAndSource. Их больше нет — не используйте.
01 ·

Что изменилось

Старая модель смешивала «сохранить триггер на компоненте» и «проиграть сейчас» в одних и тех же полях NPC Blueprint. Рефакторинг разделил ответственность: контент, реакция на мир и ручной запуск — три разных слоя.

Было

Runtime-хранилище на NPC

Квест или сканер «записывал» триггер в компонент, потом другая система «коммитила» его. Поля дублировали друг друга, легко затереть нужный тег.

Стало

Два явных API

AddBarkToQueue — событие мира. PlayBarkByTag / TriggerScriptedBark — NPC говорит по команде, минуя очередь.

Без изменений

Data Asset контента

Путь к DA_*_Barks, VO, условия, режимы выбора строк — всё на месте.

Важно

Имена в BP могли остаться

DialogAction_CommitBark, BP_QuestAction_StoreBarkData, BP_QuestAction_TriggerBark — классы с прежними именами, но внутри уже новые вызовы.

Удалено — не использовать

CommitBark, CommitBarkWithStoredData, StoreBarkData; свойства StoredBarkTrigger, BarkTrigger, StoredBarkCustomTrigger; флаг bAutoSetTargetAndSource.

02 ·

Три слоя системы

Слой 1

Контент

UBarkDefinitionDataAsset (DA_*_Barks). FBarkDefinition: TriggerResponses, Conditions, Barks, SelectionMode, Cooldown, bTriggerOnlyOnce.

Слой 2

Реактивные события

AddBarkToQueue + FBarkTrigger. Сканы, попадания, зоны, квестовый TriggerBark.

Слой 3

Скриптовый запуск

SetScriptedBark / TriggerScriptedBark или хелперы библиотеки. Задержки и циклы — на вызывающей стороне.

РЕАКТИВНЫЙ ПУТЬ
Событие → AddBarkToQueue → UBarkSubsystem → все NPC → match в DA → play

СКРИПТОВЫЙ ПУТЬ
Диалог/GAS/квест → PlayBarkByTag или TriggerScriptedBark → один NPC → direct play
Что это даёт

Зона входа и реплика охранника в диалоге — разные механизмы. Очередь: «кто-то из NPC ответит». Direct play: говорит именно этот актор.

03 ·

Словарь

Термины, которые встретятся в Editor, диалоговых графах и квестовых Blueprint.

Bark Tag/ BarkTag
Gameplay Tag, идентифицирующий тип барка: Bark.Area.Enter, Bark.Interaction.NPC.SampleResponse и т.д. Должен совпасть с одним из TriggerResponses в DA персонажа.
FBarkTrigger
Структура запроса: BarkTag, Source, Target, AdditionalGameplayTags, BlockBarksWithTags. Используется в AddBarkToQueue и при сопоставлении ответов в definition.
FBarkCustomTrigger
Настройки задержки и цикла на стороне вызывающего (DialogAction, quest BP, CustomBarkTrigger). Не хранится на NPC как «stored trigger» — передаётся в вызов или в SetScriptedBark.
Pending / Default
PendingScriptedBarkTag — runtime override от квеста или SetScriptedBark. DefaultScriptedBarkTag — значение на Blueprint NPC. TriggerScriptedBark берёт Pending, если он валиден, иначе Default.
DA_*_Barks
Data Asset с определениями барков для персонажа. Единственный источник строк, VO и правил выбора. Blueprint хранит только теги и вызовы API.
Очередь/ Queue
UBarkSubsystem (GameInstance subsystem) хранит активные триггеры и каждый тик опрашивает все зарегистрированные UBarkComponent. Подходит для событий «в мире произошло X — кто-то должен отреагировать».
Direct play
Воспроизведение на одном компоненте, минуя очередь: PlayBarkByTag, StartBarkLoop, TriggerScriptedBark. Гарантирует, что говорит именно этот NPC.

05 ·

API: BarkComponent

PlayBarkByTagDirect · немедленно

Одна строка по тегу, обходит очередь. Source/Target/AdditionalTags для conditions и FBarkResponse.

StartBarkLoop / StopBarkLoopDirect · delay/loop

Отложенное/циклическое воспроизведение с FBarkCustomTrigger.

SetScriptedBarkArm

Записывает Pending tag/timing. Не играет. Пустой BarkTag не стирает ранее armed tag — partial update для face scan.

TriggerScriptedBarkFire

Pending или DefaultScriptedBarkTag/Timing. При delay/loop → StartBarkLoop, иначе PlayBarkByTag.

07 ·

Диалоги, квесты, GAS

Dialog Graph

Класс DialogAction_CommitBark сохранил имя — в диалоговых графах ищите его как раньше. Внутри теперь PlayBarkByTag или StartBarkLoop на UBarkComponent целевого NPC. Поля настройки те же: Target NPC, Source Tag (Flow identity), BarkTag, AdditionalTags, BarkCustomTrigger, TagCooldownSeconds, InitialStartDelay.

Quest Actions

Store

BP_QuestAction_StoreBarkData

Вызывает SetScriptedBark — «заряжает» NPC для последующего срабатывания. Не проигрывает реплику сразу.

Trigger

BP_QuestAction_TriggerBark

Вызывает AddBarkToQueue — не direct play. Критично для тегов вроде Bark.Area.Enter, где важны Source/Target и fan-out.

Паттерн store + trigger

Квест Store → arm NPC (SetScriptedBark). Позже GAS или диалог → TriggerScriptedBark. Либо квест Trigger → очередь для world-reactive барков. Два разных «trigger» — не путать.

Gameplay Abilities

GA_GuardPlayBark, GA_TriggerBarkWithStoredData и подобные вызывают TriggerScriptedBark на UBarkComponent целевого NPC. Тег берётся из Pending (если квест успел arm) или из DefaultScriptedBarkTag на Blueprint guard. Типичный кейс MCRN: на BP задан Default, ability только «огонь».

Bark Trigger Area

Актор зоны (ABarkTriggerArea) при overlap игрока кладёт в очередь Bark.Area.Enter / Bark.Area.Leave с Source = игрок, Target = зона, AdditionalTags = AreaTags актора. Дизайнеру обычно достаточно настроить AreaTags на акторе и ответы в DA NPC.

Data Asset workflow

Пути

  • VO: /Game/BorderLine/Dialogs/Barks/VO/<Character>/
FBarkDefinitionОписание
TriggerResponsesFBarkResponse: BarkTag + RequiredAdditional/Source/Target tags
ConditionsInstanced Blueprint conditions
BarksVoiceOver + Subtitle
SelectionModeRandom, Sequence, SequenceLooped
Cooldown / bTriggerOnlyOnceОграничения повторов на компоненте

FBarkResponse — матчинг

Для срабатывания нужно: Trigger.BarkTag совпадает с FBarkResponse.BarkTag; AdditionalGameplayTags триггера содержат все RequiredAdditionalTags; Source/Target несут RequiredSourceTags / RequiredTargetTags (если заданы). Definitions проверяются по порядку — первый подходящий выигрывает.

11 ·

Чего не делать

✕

CommitBark / StoreBarkData

Удалены. Set+Trigger или Play/Queue.

✕

StoredBarkTrigger на BP

Не использовать runtime storage на компоненте.

✕

TriggerScripted для area

Только AddBarkToQueue с Source/Target/Additional.

✕

Текст вне DA

Контент только в DA_*_Barks.
